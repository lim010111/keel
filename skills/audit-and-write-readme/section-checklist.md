# Section Checklist by Project Type

Quick reference for which sections to include based on project type. Skip rather than placeholder — empty sections are worse than missing ones.

| Section | OSS | Personal | Internal | Config |
|---------|-----|----------|----------|--------|
| Name / Description | Yes | Yes | Yes | Yes |
| Language switch (EN ↔ KO) | Yes | Yes | Yes | Yes |
| Badges | Yes | Optional | No | No |
| Installation | Yes | Yes | Yes | No |
| Usage / Examples | Yes | Yes | Yes | Brief |
| What's Here | No | No | No | Yes |
| Architecture | Optional | No | Yes | No |
| How to Extend | No | No | Optional | Yes |
| Environment Variables | Optional | Optional | Yes | Optional |
| Contributing | Yes | Optional | Yes | No |
| Runbooks / Troubleshooting | No | No | Yes | Yes |
| Gotchas / Notes | Optional | Optional | Yes | Yes |
| License | Yes | Optional | No | No |
| Last Reviewed | No | No | Optional | Yes |

## Ordering principle

Top of the README answers "What is this?" in under five seconds. Installation and a minimal usage example come before any deep architecture. License goes last.
